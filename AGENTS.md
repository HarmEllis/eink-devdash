# Agent Instructions — eink-devdash

## Language policy

All project artifacts MUST be written in English: source code, comments, log
strings, documentation (`README.md`, `AGENTS.md`, `BOARD_NOTES.md`, etc.),
commit messages, and any in-repo notes (including `.co-develop/`).

Conversational chat with the maintainer can be in Dutch, but the moment
something lands in the repo it switches to English. If you find existing
Dutch content while editing, translate it as part of your change.

## Devcontainer is mandatory

All firmware build steps and flash-server commands MUST run **inside the
devcontainer**. Never run them directly on the host.

The devcontainer is the VS Code devcontainer named `optimistic_hermann`.
Check that it is running:

```bash
docker ps --filter "name=optimistic_hermann" --format "{{.Names}}\t{{.Status}}"
```

Run commands inside the devcontainer — **always as the `node` user** via
`-u node`:

```bash
docker exec -u node optimistic_hermann bash -c "<command>"
```

> **Important:** without `-u node` commands run as root and files (`build/`,
> `sdkconfig`, `dependencies.lock`) become root-owned, after which the
> `node` user can no longer build.

ESP-IDF is normally already on `PATH` inside the container, so a separate
`source` step is not required. If it isn't, source it explicitly:

```bash
docker exec -u node optimistic_hermann bash -c "source /etc/profile.d/esp-idf.sh && <command>"
```

## Common commands

All commands run as `docker exec -u node optimistic_hermann bash -c "..."`.

| Task | Command (inside the container) |
|------|--------------------------------|
| Build firmware | `cd /workspaces/eink-devdash/firmware && idf.py build` |
| Set target (once after clean) | `idf.py set-target esp32s3` |
| Start flash server | `cd /workspaces/eink-devdash/flash-server && bash serve.sh` |
| Install API dependencies | `cd /workspaces/eink-devdash/api && npm install` |
| Start API | `cd /workspaces/eink-devdash/api && npm start` |

## Project structure

```
firmware/          # ESP-IDF firmware (ESP32-S3)
  main/            # Application code
  components/      # Custom component: eink_weact29
flash-server/      # Web flasher (ESP Web Tools, port 8080)
  bins/            # Generated by serve.sh after idf.py build
api/               # Node.js API server (port 3000)
scripts/           # Helper scripts
```

## idf_component.yml

Do **not** add `espressif/json` to `idf_component.yml` — `json` (cJSON) is
a built-in ESP-IDF component and is pulled in only via `CMakeLists.txt`
`REQUIRES json`.

## Git safe.directory

The devcontainer runs under a different UID than the host. If git
complains, add this once:

```bash
docker exec optimistic_hermann git config --global --add safe.directory /workspaces/eink-devdash
```
